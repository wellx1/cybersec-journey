## Learned

**Advanced permissions:**

| Command/Concept | What it does |
|----------------|-------------|
| `umask` | Sets default permissions for newly created files |
| `chmod u+s file` | Sets SetUID - file runs as owner not the executor |
| `chmod g+s file` | Sets SetGID - file runs as group or inherits group in directories |
| `chmod +t directory` | Sets Sticky bit - only file owner can delete inside directory |

**SetUID recap:**
When set on a file owned by root, anyone who runs it
temporarily gets root permissions for that execution only.
Classic privilege escalation target in CTFs and pentests.

**SetGID recap:**
On files works like SetUID but for group permissions.
On directories new files created inside inherit the
directory group instead of the creator's group - useful
for shared team directories.

**Sticky bit:**
Commonly seen on `/tmp` directory. Anyone can create
files there but only the owner of each file can delete
their own files. Prevents users from deleting each
other's temporary files.

**Umask:**
Default permissions for new files are set by umask.
Umask subtracts from the maximum permissions:
- Default max for files: `666`
- Default max for directories: `777`
- Common umask `022` results in `644` for files and `755` for directories

**Bandit Level 12 - solved:**

Most complex level so far. Started with a hexdump file
that had to be reconstructed and decompressed through
multiple layers. Commands used:

| Command | What it did in this level |
|---------|--------------------------|
| `mktemp -d` | Created a temporary working directory |
| `cp` | Copied the file to the temp directory |
| `xxd -r` | Reversed the hexdump back to binary |
| `file` | Identified the compression type at each layer |
| `gunzip` | Decompressed gzip layers |
| `bunzip2` | Decompressed bzip2 layers |
| `tar -xf` | Extracted tar archive layers |
| `mv` | Renamed files with correct extensions before decompressing |

The level required identifying the compression format
at each layer with `file`, renaming the file with the
correct extension, decompressing it, then repeating
the process multiple times until reaching plain text.
