## Learned

**Directory commands:**

| Command | What it does |
|---------|-------------|
| `mkdir foldername` | Creates a new directory |
| `mkdir -p folder/sub/subsub` | Creates nested directories in one command |
| `rmdir foldername` | Deletes a directory only if completely empty |
| `rm -r foldername` | Deletes a directory and everything inside it |
| `rm -rf foldername` | Forces deletion with no confirmation prompts |

**Move and rename commands:**

| Command | What it does |
|---------|-------------|
| `mv file destination` | Moves a file to another location |
| `mv oldname newname` | Renames a file |
| `mv -i file destination` | Asks for confirmation before overwriting |
| `mv -v file destination` | Verbose - shows what is happening |
| `mv -b file destination` | Creates a backup before overwriting |

**Getting help:**

| Command | What it does |
|---------|-------------|
| `help` | Shows built in shell commands |
| `command --help` | Shows flags and usage for any command |

**Key insight on rm:**
Using `rm -i` is the safer habit for beginners - it asks
for confirmation before each deletion so you never
accidentally remove something important. Once comfortable
with the command `rm -rf` is faster but requires more
confidence that you have the right target.

**Key insight on rmdir vs rm -r:**
`rmdir` is actually the safer choice for deleting directories
because it refuses to delete anything if the folder still
has content inside - acts as a natural safety net against
accidental deletion of important files.

## Struggles
`mkdir -p` was throwing a permission denied error when using
`/foldername/foldername2` at the start of the path.

Root cause - `/` at the beginning tells Linux to start from
the root of the entire system which is protected and requires
admin permissions. Simple fix was removing the leading slash
so the path starts from the current location instead.

**Rule to remember:**

| Path start | Where it goes |
|------------|--------------|
| `/foldername` | Root of the system - protected |
| `~/foldername` | Your home directory - full access |
| `./foldername` | Your current location - full access |
| `foldername` | Your current location - full access |

## Cybersecurity Connection
The `-i` flag pattern exists across many dangerous Linux
commands as a safety mechanism. Understanding which commands
are destructive and irreversible is fundamental security
hygiene - both for protecting your own systems and for
understanding what an attacker can do on a compromised
machine. A single `rm -rf` pointed at the wrong directory
can destroy an entire server in seconds with no recovery.
