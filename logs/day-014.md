## Learned

**User Management:**

| Command | What it does |
|---------|-------------|
| `sudo command` | Runs a single command as superuser |
| `su username` | Switches to another user session entirely |
| `useradd username` | Creates a new user |
| `userdel username` | Deletes a user |
| `usermod` | Modifies an existing user |
| `groupadd groupname` | Creates a new group |
| `groupdel groupname` | Deletes a group |
| `passwd username` | Changes a user password |

**Where users and groups are stored:**

| File | What it contains |
|------|-----------------|
| `/etc/passwd` | All users on the system |
| `/etc/shadow` | Encrypted user passwords |
| `/etc/group` | All groups and their members |

Key insight on `sudo` vs `su` - `sudo` is the correct
practice for running privileged commands. It runs only
that specific command as root then drops back to normal
user. `su` switches your entire session to another user
which is considered bad practice because it gives broader
access than necessary and leaves less audit trail.

**Encoding commands learned through Bandit:**

| Command | What it does |
|---------|-------------|
| `base64 file.txt` | Encodes file content to base64 |
| `base64 -d file.txt` | Decodes base64 encoded content |
| `tr 'A-Za-z' 'N-ZA-Mn-za-m'` | Applies ROT13 cipher to text |

**Bandit Level 10 - solved:**

File contained base64 encoded content. Had no previous
knowledge of base64 so read the man page, tested encoding
to understand the direction, then applied the decode flag:

```bash
base64 -d data.txt
```

Key insight - base64 is not encryption, it is encoding.
It is reversible by anyone without a key which is why
finding it in a real system is a red flag. Often used
by attackers to obfuscate malicious commands or payloads.

**Bandit Level 11 - solved:**

Password encoded with ROT13 cipher - each letter shifted
13 positions forward in the alphabet. The syntax for
applying ROT13 using `tr` was the hardest part:

```bash
cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m'
```

The confusing part was understanding why the alphabet
had to be split into two ranges. ROT13 shifts 13
positions but the alphabet only has 26 letters so after
Z it wraps back to A. The `tr` command cannot wrap
automatically so the two ranges handle each half:
- `A-M` maps to `N-Z`
- `N-Z` maps to `A-M`
- Same logic repeated for lowercase

Spent significant time on the Wikipedia ROT13 page until
the wrap around concept clicked. Worth every minute.
