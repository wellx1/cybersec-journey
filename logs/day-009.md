**Text-fu - Standard streams:**

| Concept | What it is |
|---------|-----------|
| `stdout` | Standard output - where commands send their results |
| `stdin` | Standard input - where commands receive their input |
| `stderr` | Standard error - where error messages go separately |

**Redirection operators:**

| Operator | What it does |
|----------|-------------|
| `>` | Redirects output to a file, overwrites if exists |
| `>>` | Redirects output to a file, appends if exists |
| `<` | Feeds a file into a command via stdin |
| `2>` | Redirects error messages to a file |

**Key insight on stdout:**
`echo "text" > file.txt` does not just write text - it
redirects the output stream from the terminal into a file.
The terminal is the default destination of all output,
redirection just changes where that output goes.

**Key insight on stdin:**
`cat < file.txt` and `cat file.txt` produce the same result
but work differently internally. With the argument the
command opens the file itself. With `<` the shell opens
the file and feeds the contents to the command directly.
The difference becomes important with commands that do
not accept filename arguments.

**Key insight on stderr:**
Errors go to a separate stream from normal output on
purpose - so you can redirect errors and results
independently. `2>` captures only errors leaving normal
output untouched.
