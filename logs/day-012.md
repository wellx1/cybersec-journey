## Learned

**Text manipulation commands:**

| Command | What it does |
|---------|-------------|
| `tail -n 5 file.txt` | Shows last 5 lines of a file |
| `tail -f file.txt` | Follows file in real time, updates as new content is added |
| `expand file.txt` | Converts tabs to spaces |
| `unexpand file.txt` | Converts spaces back to tabs |
| `join file1.txt file2.txt` | Merges two files based on a common field |
| `split -l 100 file.txt` | Splits a large file into smaller chunks of 100 lines each |

Key insight on `tail -f` - extremely useful for monitoring
log files in real time. As new entries are written to the
file the terminal updates instantly without needing to
rerun the command.

**Bandit Level 5 - solved:**

Looking for a file that was human readable, exactly 1033
bytes and not executable across multiple directories.
Built the command step by step using the find manual:

```bash
find . -type f -size 1033c ! -executable | xargs file | grep ASCII
```

Broke down the logic:
- `find . -type f` - find all files from current location
- `-size 1033c` - exactly 1033 bytes
- `! -executable` - not executable
- `| xargs file` - check file type of results
- `| grep ASCII` - filter for human readable only

**Bandit Level 6 - solved:**

Looking for a file owned by a specific user, specific
group and exactly 33 bytes stored anywhere on the server.
Had not studied users and permissions yet so read the
`find` manual directly in the terminal using `man find`
and figured out the flags independently:

```bash
find / -user someuser -group somegroup -size 33c 2>/dev/null
```

Key discoveries:
- `/` searches the entire server not just current directory
- `-user` and `-group` flags exist in find for ownership filtering
- Command returned masses of permission errors initially
- Applied `2>/dev/null` to redirect all errors to nothing
- One clean result appeared with the password file location
- `cat` on the full path returned the password
