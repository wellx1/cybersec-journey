**New commands practiced:**

| Command | What it does |
|---------|-------------|
| `history` | Shows every command you have typed in the terminal |
| `history -p` | Clears history in zsh (not history -c which is bash only) |
| `fc -p` | Alternative way to clear history in zsh |
| `rm ~/.zsh_history` | Deletes the history file from disk permanently |
| `cp file destination` | Copies a file to another location |
| `cp -r folder destination` | Copies an entire folder and its contents |
| `mv oldname newname` | Renames a file |
| `mv file ~/destination` | Moves a file to a different directory |

**Key insight on cp vs mv:**
`cp` keeps the original and creates a copy at the destination.
`mv` removes the original and places it at the destination.
One duplicates, the other relocates.

## Struggles
Running zsh instead of bash on Kali caused the first real
diversion from the course material. `history -c` is a bash
command and does not work in zsh, which threw an error and
caused confusion.

Resolved by identifying the active shell with `echo $SHELL`
and finding the zsh equivalent commands. Good lesson that
tutorials are not always written for your exact setup and
independent problem solving is part of the process.

**Shell differences to remember:**

| Thing | bash | zsh |
|-------|------|-----|
| Clear history | `history -c` | `fc -p` |
| History file | `~/.bash_history` | `~/.zsh_history` |
| Config file | `~/.bashrc` | `~/.zshrc` |

## Cybersecurity Connection
Understanding shell differences matters in real security work.
When you gain access to a machine during a pentest you need
to immediately identify which shell is running with
`echo $SHELL` because commands and scripts behave differently
depending on the answer. Attackers also clear bash and zsh
history to cover their tracks after compromising a system -
knowing where history lives and how to clear it is both an
offensive and defensive skill.
