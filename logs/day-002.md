## 💡 Learned

| Command | What it does |
|---------|-------------|
| `echo "text"` | Prints whatever text you put between quotes back to the terminal |
| `whoami` | Returns your current username |
| `id` | Shows full identity — user ID, group ID and system privileges |
| `id -un` | Returns just the username, same output as whoami |
| `pwd` | Shows your current location in the filesystem |

**Key insight:** `whoami` and `id -un` return the same output
but `id` is a much more powerful command with many variations.
`whoami` exists only to print the username — `id` can show
full privilege information which is critical during pentesting.

## 🔐 Cybersecurity Connection
`id` is one of the first commands run after getting access to
a machine during a pentest or CTF. If the output shows
`uid=0(root)` you have full control of the system. Understanding
user privileges is fundamental to both attacking and defending
systems — privilege escalation (moving from normal user to root)
is one of the most common attack techniques you will study later.
