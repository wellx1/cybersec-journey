## Learned

**Pipe operator:**

| Concept | What it does |
|---------|-------------|
| `command1 \| command2` | Takes output of command1 and feeds it as input to command2 |
| `command1 \| command2 \| command3` | Chains multiple commands, each refining the previous output |

Key insight - the pipe intercepts output before it hits
the screen and redirects it silently into the next command.
Instead of seeing everything, you only see what the last
command in the chain produces.

**tee command:**

| Command | What it does |
|---------|-------------|
| `command \| tee file.txt` | Sends output to screen AND saves it to a file simultaneously |
| `command \| tee -a file.txt` | Same but appends instead of overwriting |

Key insight - tee does two things at once. Without tee
you have to choose between seeing output on screen or
saving it to a file. tee removes that choice.

**Environment variables:**

| Command | What it does |
|---------|-------------|
| `env` | Lists all current environment variables |
| `VAR=value` | Creates a local variable for current session only |
| `export VAR=value` | Creates an environment variable available to all processes |
| `echo $VAR` | Reads and prints a variable value |

Key insight - local variables only exist in the current
shell session. Environment variables are passed down to
any process or command spawned from that session. That
is why `export` matters - without it other programs
cannot see your variable.
