## Learned

**Commands:**

| Command | What it does |
|---------|-------------|
| `touch filename` | Creates an empty file |
| `touch -d "date" file` | Creates file with a specific timestamp |
| `touch -r file1 file2` | Copies timestamp from file1 to file2 |
| `file filename` | Shows the actual file type - txt, png, jpg, mp4 etc |
| `cat filename` | Reads and prints file content to terminal |
| `cat file1 file2` | Reads multiple files at once in sequence |
| `less filename` | Opens file in viewing mode - useful for long files |

**Key insight on `less`:** much more practical than `cat` for
long files because you can scroll. Once inside use `/word`
to search for a specific term, `q` to quit.

**Key insight on `file`:** Linux does not trust file extensions
the way Windows does. A file called `image.jpg` might actually
be something else entirely. `file` reads the actual content
to tell you what it really is - important habit in security work.

## Lab Practice - OverTheWire Bandit

**Level 0 solved:**
Used `ls` to find the readme file then `cat readme` to read
the password. Straightforward introduction to basic navigation
and file reading.

**Level 1 solved:**
File was named `-` which caused problems because Linux reads
dash as a flag, not a filename. Tried `cat -` and it didn't
work. Found the fix by using `cat ./-` where `./` explicitly
tells Linux this is a file in the current directory, not a flag.

This was the real lesson of Level 1 - weird filenames need
special handling. Pattern to remember:

| Weird filename | How to handle it |
|---------------|-----------------|
| Named `-` | Use `cat ./-` |
| Has spaces | Use `cat "file name"` or `cat file\ name` |
| Starts with `.` | Hidden file, use `ls -a` to find it first |

---

## Cybersecurity Connection
The `file` command is used constantly in forensics and CTFs.
Attackers often rename malicious files to look harmless -
a malware executable renamed to `document.pdf` fools a
double click but not `file document.pdf` which will expose
what it actually is. Never trust a file extension, always
verify the real type.
