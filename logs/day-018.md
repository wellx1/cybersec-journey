## Learned

**Bash script basics:**

| Concept | What it means |
|---------|--------------|
| `#!/bin/bash` | Shebang line - tells system to use bash interpreter |
| `chmod +x script.sh` | Makes script executable |
| `./script.sh` | Runs the script from current directory |
| `bash script.sh` | Runs script explicitly with bash regardless of shebang |

**Key insight on shebang vs active shell:**
The shebang and your active terminal shell are independent.
Running zsh as your terminal does not stop you from writing
and running bash scripts. The shebang tells the system
which interpreter to hand the script to - they do not
need to match.

**Common mistake learned:**
Writing `#!/bin/zhs` instead of `#!/bin/bash` - typo in
the shebang causes `./script.sh` to fail with bad
interpreter error while `bash script.sh` still works
because bash is called explicitly.
